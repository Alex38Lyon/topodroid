[8] SURVEY DATA WINDOW

Back to the main window. The list of surveys is empty, but you are finally ready for your first survey. Tap on the Add button (the "plus" sign). You get a dialog to specify the survey information: name, date, team, initial survey station, magnetic declination, and a brief description.

The survey name is mandatory. It must be unique in the database. The survey names are used to list surveys in the main window and TopoDroid distinguishes surveys by their name (although each survey has a unique numerical id in the database). The date is pre-filled with the current date. Team and description are empty (unless you have set a default team in the app settings). They can safely be left so, although it is a good practice to take the time to fill them. The magnetic declination is 0.0 by default. If you know it you can enter it here, otherwise you can look it up in the web after you have found the coordinates of a fixed point (say, the cave entrance) with the GPS [A].

WARNING.
The name of the survey cannot contain the character '/' (slash), which is the pathname separator.
The date must be written with format "yyyy.mm.dd". For example January 6, 2015 is "2015.01.06".

Save when you are done, and the new survey name will appear in the list of surveys. A long tap on a survey name opens the survey info dialog again. You can change any information. The survey name is special, because once the survey has been committed to the database, the name is used to refer to it. There is a menu to modify the survey name.

If you tap shortly on the survey name the survey data activity starts. This is one of the two most important activities of TopoDroid. The other is the Sketch activity.

TopoDroid differentiates among four kinds of shots:

  1. "blank" shots, ie, shots with the stations not set yet
  2. "splay": it has only one station, usually the "From" station
  3. "leg" (the first shot of a group of repeated measurements): this shot has both the "From" and the "To" station set
  4. "repeated-leg" (the following shots of a group of leg shots): these shots do not need to have the stations set

Each kind of shots is displayed with a different color: "legs" are white, "repeated-legs" grey, "splays" blue, "blank" red. Furthermore the background color is blueish for the last downloaded shots, and reddish for "unreliable" shots, ie. shots with "abnormal" field intensities (only DistoX2).

The "reliability" of a shot is estimated by comparing the intensities of the magnetic and gravitational fields and the dip angle of the magnetic field with their averege values, respectively. If any of these differs from the average by more that a specified threshold, the shot is deemed "unreliable". The thresholds are among the app settings.

MENUS

  • Survey info: to open the survey information window
  • Recover: to recover a deleted item (shot or sketch)
  • Photo [A]: lists the pictures taken for this survey
  • Sensor [A]: lists the sensor data taken for this survey
  • 3D: to display the survey in 3D (need the app Cave3D)
  • DistoX [A]: opens the Device window
  • Settings
  • Help

BUTTONS

  • Download: it will download the data from the DistoX as in the calibration data window. If no DistoX is selected this button is hidden.
  • Bluetooth: Opens the Bluetooth dialog that allows you to control the laser, take a measurement and download it, and reset the BT connection. If you experience problems with the DistoX after downloading some data, you may try to reset the bluetooth connection on TopoDroid, switch the DistoX on and off, and retry downloading the data.
  • Display: to choose the display mode (see below)
  • Sketch: opens the Sketch dialog where you can ask the program to create a new sketch or open an existing one. The dialog has a button to create a new sketch, and a two-column list of the current survey sketches (items in the left column open a sketch with its plan view, the right column items open sketches in profile view; once open in a view you can switch to the other from within the sketch window). If the 3D modelling is enabled there is also a button to create a new 3D model, and the existing 3D models are listed after the sketches. A long tap on the Sketch button opens immediately the most recent sketch, without going through the sketch list dialog.
  • Notes opens the dialog to edit the annotations of the survey
  • Extend opens the "Extend reference" dialog that defines how to set the "extend" for the coming shots (either downloaded or manually entered).
  • Stations [N]: this buttons opens the dialog for the saved stations
  • Add [N] adds a shot. In the coming dialog you must enter the shot stations and data (length, azimuth, and clino). You can specify also the direction of "extend", and values for LRUD. By default these are assumed at the FROM station, unless you check the box for the TO station.

SURVEY SETTINGS

  • Team: the default surveying team. If the value is not empty, it is used to set the "Team" field when you create a new survey. You can write your name and that of the cavers you usually survey with.
  • Stations policy: this tells the program how to automatically assign survey stations to the shots. (This is further explained below).
  • Station names specifies the type of the station names, either alphanumeric (default) or numeric [B].
  • Initial station name: the default first station name for new surveys. Can be also changed in the New survey dialog.
  • Thumbnail size: size of the photo thumbnails
  • DATA UNITS >>
    • Length units: either meters or feet
    • Angle units: either degrees or grads
  • SHOT DATA >>
    • Leg tolerance: the angular tolerance (percent) between consecutive shots to belong to the same leg
    • Minimum leg shots: minimum number of close consecutive shots that are takes as a leg. Further consecutive close shots are taken as belonging to the leg as well.
    • Splay vertical threshold: the "extend" direction assigned to a splay shot depends on the angle it makes with the leg(s). When the angle is closer than this value to 90 degrees the splay gets a "vertical extend". Otherwise it gets the same extend as the leg, or the opposite, depending whether the angle it makes with the leg is smaller or larger than 90 degrees.
    • Leg vertical threshold. this is like Therion "vthreshold": the LRUD for manually entered shots are considered West-East-North-South when the shot inclination exceeds the vertical threshold
    • Settable extend reference. The L/R extend reference can be either fixed (180 degrees) or adjustable.
    • Loop closure: whether to do the loop closure error compensation. It is advisable not to do the compensation, as this allows to see graphically the misclosures, and spot possible survey errors
    • Backsight: whether to add a line with fields for the backsight in the shot manual input dialog [N].
    • Magnetic anomaly: whether to compensate the local magnetic anomaly. It requires backsight auto-station policy.
    • Timer is the number of seconds before measuring the direction with the Android sensors (for "manual data-input" dialog [N], and for the "extend-reference" dialog))
    • Beep volume is the volume of the timer beep [N]
  • LOCATION >>
    • Location units: either dd.mm.ss or decimal degrees [A]
    • C.R.S. coordinate reference system [default is long-lat] (disabled)
  • ACCURACY settings

DISPLAY MODE

The Display button opens a dialog with choices for "shot id", "splay shots", "blank shots", and "repeated leg shots". The "shots id" are unique increasing indices assigned by TopoDroid to the shots of the survey. Splay shots can be shown or hidden. Blank shots can be either all shown or hidden. The same for repeated-leg shots.

If the splay shots are hidden it is still possible to show the splay shots for an individual station. Tapping on the station name in a leg shot, makes the splay shots at that station visible. Tapping again on the station name (even in a splay shot) hides them.
