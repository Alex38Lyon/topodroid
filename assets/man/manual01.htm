[1] INTRODUCTION

TopoDroid has a main window from which you can move to other app activities. The main activities are:
  • Survey data management (survey and shot-list windows).
  • Sketch drafting (drawing window) is a sub-activity of the Survey activity.
  • DistoX functions (device window)
  • Calibration of DistoX (calibration and calibration-data windows) is a sub-activity of the Device activity.


Topodroid windows have a scrollable button bar at the top, for the most frequent actions, and a menu button at the top-right. The dialogs usually have a confirmation button (OK or Save). If you want to close the dialog, or cancel whatever you have done, tap the Android BACK button. If there are both the OK and the Save button, the former saves the input data and closes the dialog, while the latter saves the input data and keep the dialog open for a new input. Information dialogs have no buttons.

In every TopoDroid activities the hardware MENU key opens the Settings dialog.

Here is some background about how the survey data, the calibration data, and the sketches are stored. You can safely skip the following and the next two sections, and come back to them later if needed.

The app creates its default base directory, named "TopoDroid", in the primary external memory (usually /sdcard). This directory is not removed when you uninstall the app. In this directory the app creates two SQLite databases. The first is "device10.sqlite", where the calibration data, the DistoX informations, and other data are stored. The second is "distox14sqlite" and stores the surveys data. The app also creates subdirectories for the different types of files:

  • bin: DistoX2 firmware files
  • csv: CSV text files (exported surveys and calibrations)
  • csx: cSurvey files (exported surveys and sketches)
  • dat: Compass files (exported surveys)
  • dxf: DXF files (exported surveys and sketches)
  • dump: DistoX memory dump files
  • import: survey files to import (Therion, Compass, VisualTopo, and PocketTopo format)
  • kml: GoogleEarth exports [A]
  • note: text note files (survey annotations)
  • photo: survey pictures (each survey with its own subdirectory)
  • plt: OziExplorer track file exports [A]
  • png: PNG image files (exported sketches)
  • srv: Walls files (exported surveys)
  • svg: SVG files (exported sketches)
  • svx: Survex files (exported surveys)
  • symbol with the drawing tools files (points, lines, areas)
  • th: Therion files (exported surveys)
  • th2 for Therion scrap files (saved sketches)
  • th3 for 3D model files
  • tmp: for temporary files. Any file leftover in this directory can be safely deleted.
  • top: PocketTopo files (exported surveys and sketches)
  • tro: VisualTopo files (exported surveys)
  • zip: survey ZIP archives (exported and importable)

Some of these subdirectories are used only to store exported files (csx, dat, dxf, png, svx, th, top, tro). Others are used to store data (th2, th3, note, photo). The subdirectory "symbol" contains data files used by the program. The subdirectory "zip" is used to store zipped archives as well to read them and uncompress in TopoDroid. When you want to import a survey made with another program (Therion, Compass, VisualTopo, or PocketTopo) you must place it in the subdirectory "import". Finally "bin" is used to store firmware files, both the factory files included in TopoDroid and the firmware dumps read from a DistoX.

Current Work Directory

TopoDroid does not manage complex survey projects. However it is possible to specify a different current work directory (CWD) for the data files. Each CWD is a subdirectory of the primary external memory, and it is created if it does not exists. The name of the current work directories must begin with "TopoDroid".

You can use different directories for different cave survey projects. The firmware files and the drawing symbol files are installed only in the default directory. Alternatively you can have one "TopoDroid" directory for each project and switch to the one you must work on when you need by renaming directories with a file manager. However in this way the common utility files are replicated.

The choice of the current work directory is one of the general settings of the app.

Language

By default TopoDroid uses the system default language, if it contains the proper translation, or English. You can override this choice selecting another language among the supported ones: English, Bulgarian, French, German, Italian, Russian, and Spanish.

The choice of the language is one of the general settings of the app.

Text and Buttons size

The "Text Size" setting specifies the dimention of the list text entries, in particular the list of surveys, that of shots, and that of calibration data.
There are two choices for the size of buttons: "small" and "large". The first is suitable for cellphones (5' screen), and is the default. The second is suitable for tablets (7' screen).

Custom Keypads

TopoDroid comes with a numerical custom keypad and a simple "qwerty" keyboard. The numerical keypad is used for numerical values (lengths, angles, etc.). It has the ten digits, the decimal point, and the plus/minus sign. For the input of coordinates it has also the "degree" ('d') and "minute" ('m') keys.

The querty keyboard is used for station names. It has the ten digits, the 26 letters of the latin alphabet (lowercase and uppercase), and the period ('.').

The custom keypads are enabled by default. They can be enabled/disabled through a settings of the main TopoDroid window.
