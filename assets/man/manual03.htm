[3] DEVICE WINDOW

Before surveying you need to connect and pair the DistoX with the Android and calibrate it. Tap the DistoX button from the Main windows to open the Device window. The list of known DistoX devices s shown in the middle. TopoDroid works with one DistoX at a time. To select it tap on its entry in the list. The name of the selected DistoX is shown at the top, just below the buttons. If no DistoX is selected, there is a red message saying "Device not selected".

If the DistoX list is empty, your DistoX ineed to be connected and paired with the Android. This can be done with TopoDroid (Scan and Pair menus) or with the Settings app.

DistoX nickname
By default TopoDroid uses the BT address as name for a DistoX. However you can assign a nickname to the DistoX to make it easier to distinguish among devices if you work with several of them. Long tap on a device to enter its nickname.

ACTIONS

  • tap on a DistoX makes it the current DistoX
  • long tap on a DistoX sets the nickname

BUTTONS

You need the DistoX turned on for button actions, except for the first and third buttons.
  • Bluetooth connection reset. No connection with the DistoX is established.
  • Mode-toggle toggles the DistoX calibration mode on and off
  • Calibrations of the current device.
  • Memory [A] DistoX memory functions
  • Read [A] reads and displays the calibration coefficients stored in the DistoX.
  • Info [A] reads some DistoX information (hw/fw, etc.). This dialog has also two checkboxes through which you can force the DistoX model to TopoDroid: select the model and press the "save" button. Beware that if you set the wrong model some device function might not work

Calibrations dialog
The Calibrations dialog displays the list of calibrations of the current device, that are recorded in the database. If no calibration is found the list is empty. A new calibration is added with the "New Calibration" button. The dialog has also a button to import a calibration from the exported csv file.

MENUS

  • Scan searches for new DistoX devices. The DistoX must be on. You might have to try two or three times before the DistoX is found.
  • Pair: pairs the current DistoX device with the Android, if not already paired
  • Disconnect detaches the current device. The pairing between Android and the DistoX is not broken
  • Firmware [A]: used to dump and upload firmwares to the DistoX2
  • Settings: device activity settings
  • Help

DEVICE SETTINGS

  • Bluetooth: whether to enable, disable, or check on start-up [default]
  • Data connection: either stay connected to the DistoX to download shot data as soon as they are taken, or download the shots in burst, "on-demand" [default]. This applies only to data: other DistoX functions use a one-shot query mode
  • Auto reconnect: automatically reconnect if the connection is lost in "continuous" data download-mode
  • Socket type: either normal [default] or "insecure"
  • Z6-workaround Fixes a download timeout error on a Z6 phone, and possibly other devices. It is safe to leave this enabled
  • Connection delay: delay [1/10 sec] before trying to connect to the DistoX [default 0, no delay]
  • Auto pairing: whether TopoDroid should try to pair with the DistoX selected from a scan
